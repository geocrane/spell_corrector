# Corrector

Приложение для автоматизированной проверки орфографии и грамматики в документах Microsoft Word и Outlook с использованием современных технологий машинного обучения (LLM).

## Основные возможности

- **Интеграция с Microsoft Office**: Автоматический поиск всех открытых документов Word и писем Outlook через COM-интерфейс.
- **Интеллектуальная проверка**: Использование модели T5 (`sage-fredt5-distilled-95m`) с PEFT-адаптерами для качественного исправления ошибок.
- **Визуальное сравнение (Diff)**: Наглядное отображение изменений (удаления и вставки) с цветовым кодированием.
- **Сохранение форматирования**: Применение исправлений непосредственно в документ с сохранением исходного стиля текста.
- **Гибкая настройка**: 
    - Строгая защита специальных токенов (числа, аббревиатуры, валюты).
    - Аудиторский формат (автоматическая нормализация сокращений и символов).
    - Работа с таблицами (возможность пропуска или включения).
    - Пользовательский блок-лист слов.

## Требования

- **ОС**: Windows (необходимо для работы с Microsoft Office через COM-объекты).
- **Python**: 3.8 или выше.
- **Microsoft Office**: Установленные Word и/или Outlook.

### Зависимости

Для работы приложения требуются следующие библиотеки:
- `torch`
- `transformers`
- `peft`
- `pywin32` (модули `win32com`, `win32gui`)

## Установка

1. Склонируйте репозиторий:
   ```bash
   git clone <url-репозитория>
   cd corrector_1.5
   ```

2. Установите необходимые зависимости:
   ```bash
   pip install torch transformers peft pywin32
   ```

3. Убедитесь в наличии модели и адаптеров:
   - Модель должна находиться в папке: `model/sage-fredt5-distilled-95m/`
   - Адаптеры должны находиться в папке: `adapters/` (например, `adapters/lora_adapter/`)

## Использование

1. Запустите основной скрипт:
   ```bash
   python main.py
   ```

2. В открывшемся окне:
   - Выберите нужный документ из списка открытых.
   - Нажмите "Проверить весь документ" или выделите фрагмент текста в Word и нажмите "Проверить выделенный фрагмент".
   - Изучите предложенные исправления в виде карточек.
   - Нажмите на карточку, чтобы применить исправление в документе, или используйте кнопку "Применить все".

## Конфигурация (`config.json`)

Вы можете настроить поведение приложения через файл `config.json`:
- `default_adapter`: Адаптер, используемый по умолчанию.
- `strict_protection`: Если `true`, приложение будет более осторожно относиться к изменению специфических терминов и чисел.
- `auditor_format`: Включает замену общепринятых сокращений на полные формы или специальные символы (например, `руб.` -> `₽`).
- `word_blocklist`: Список слов, которые модель не должна изменять.

## Логирование

Все действия и ошибки записываются в файл `spell_debug.log`, что помогает в диагностике проблем при работе с документами или моделью.
